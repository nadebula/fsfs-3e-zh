# 自由软件现在变得更加重要

自 1983 年始，自由软件运动一直为计算机用户的自由而奔走——为了用户能够控制他们所使用的软件，而非与之相反。如果某个程序尊重用户的自由和社区，我们称之为“自由软件”。

我们有时也称之为“libre 软件”以强调我们是在谈论自由而非价格。诸如 Photoshop 等某些私有（非自由）软件非常昂贵；而诸如 Flash 播放器等其他一些则可以免费获得——但这只是次要细节。无论采用何种方式，它们赋予了该程序的开发者凌驾于用户之上的权力，一种无人应当拥有的权力。

这两种非自由软件拥有某些其他共同点：它们都是**恶意软件**。也就是说，它们都拥有被设计为虐待用户的功能。如今的私有软件通常是恶意软件，由于其开发者的权力腐化了它们[^1]。通过自由软件，用户控制程序，既包括个人控制也包括集体控制。于是他们能够控制他们的计算机做什么事情（假设这些计算机是忠诚的，并且将会做用户的程序告知它们去做的事情）。

通过私有软件，程序控制用户，而某些其他实体（开发者或者“拥有者”）控制该程序。于是该私有软件赋予了其开发者凌驾于其用户之上的权力。这就其本身而言是一种不公，并且诱使开发者通过其他方式虐待用户。

自由意味着拥有对您自己的生活的控制权。如果您使用某个程序来执行您的生活中的行为，您的自由依赖于您拥有对于该程序的控制权这一点。您理应拥有对于您所使用的程序的控制权，如果在您将其用于您的生活中的某些重要事情的时候是这样则更好。

用户对于程序的控制要求四项基本自由[^2]。

0. 出于任何目的如您所愿运行该程序的自由。
1. 研究该程序的“源代码”并对其进行修改的自由，以使得该程序如您所愿进行您的计算。程序由程序员使用某种编程语言编写——就像英语和代数相结合——程序的这种形式称为“源代码”。任何了解编程，并且以源代码的形式拥有该程序的任何人都可以阅读源代码，了解其作用，并且对其进行修改。如果您所得到的全部只是可执行文件的形式，一系列计算机可以高效地运行但是人类极难理解的数字，理解并且修改这种形式的程序如同被禁止般困难。
2. 如您所愿制作并且分发完整副本的自由。（这不是一种义务；是否如此做是您的选择。如果某个程序是自由的，这并不意味着某人有义务为您提供一份副本，或者您有义务为此人提供一份副本。向用户分发程序而不随之提供自由的行为虐待了用户；然而选择不分发该程序——对其进行私人使用——并未虐待任何人。）
3. 如您所愿制作并且分发您的修改版本的自由。

前两项自由意味着用户可以行使对于该程序的个人控制。通过另两项自由，任何用户团体可以行使对于该程序的**集体控制**。通过全部四项基本自由，用户能够完全控制该程序。如果它们中的任何一者缺失或者不充分，该程序为私有（非自由）的，并且是一种不公。

其他类型的作品也可用于实践活动，包括烹饪菜谱、诸如教科书等教育作品、诸如字典和百科全书等参考作品、用于显示文本段落的字体、供人们进行构建的硬件的电路图、利用三维打印机制造有用的（不仅仅是装饰性的）物体的结构等等。由于这些并非软件，自由软件运动严格地说并未覆盖它们；但是相同的理由同样适用，并且得出相同的结论：这些作品应当携带四项基本自由。

自由软件允许您对其进行修补，以使其做那些您想要做的事情（或者停止做某些您不喜好的事情）。修补软件可能听起来有些荒唐，如果您习惯于作为一个密封盒子的私有软件，但是这是自由世界中可做的一件普通的事情，以及学习编程的一种好方法。甚至是传统美国人的修补汽车的业余爱好也受到了阻碍，由于现在的汽车包含非自由软件。

## 私有性的不公

如果用户不能控制程序，则该程序控制用户。对于私有软件，总有一些实体，诸如该程序的开发者或者“所有者”控制该程序——并且通过它行使其凌驾于用户之上的权力。非自由软件是一种枷锁，一种实施不公权力的工具。

在令人不可容忍的案例中（尽管这种愤怒已经变得相当平常），私有软件被设计为监控、限制、审查以及虐待用户[^3]。例如苹果 iThing 的操作系统做了所有这些，而带有 ARM 芯片的移动设备上的 Windows 也是如此。Windows、移动电话固件，以及 Windows 版本 Google Chrome 包含通用后门以允许某些公司远程更改程序而无需请求许可。亚马逊焚书机（Kindle）拥有可用于删除书籍的后门。

“物联网”中的非自由软件的使用将会使其成为“电话推销员之网”[^4]以及“窃听者之网”。

带着终结非自由软件的不公这一目标，自由软件运动开发了自由软件，于是用户可以解放自己。我们自 1984 年始开发自由操作系统 GNU。今天，有千百万台计算机运行 GNU，主要是通过 GNU/Linux 的结合[^5]。

向用户分发某个程序而不为其提供自由虐待了那些用户；然而，选择不分发该程序并未虐待任何人。如果您编写了一个程序并且私人使用，这并未对他人作恶。（您确实错失了行善的机会，但是这并不等同于作恶。）因此，当我们说所有软件都必须自由的时候，我们指的是每一份副本都必须带有四项基本自由，但是我们并不是在说某人有义务为您提供一份副本。

## 非自由软件与 SaaSS

非自由软件是商业公司控制人们的计算的第一种方式。如今，还有另一种方式，称为“作为软件替代品的服务”（SaaSS）。这意味着使得某些其他人的服务器执行您自己的计算任务。

SaaSS 并不意味着服务器上的程序为非自由的（尽管它们通常如此）。与之相反，使用 SaaSS 导致了与使用非自由软件相同的不公：通往同一个坏去处的两条路径。以 SaaSS 翻译服务为例：用户将文本发送至服务器，服务器翻译它（例如从英语到西班牙语）并且将译文发送回用户。现在，这项翻译任务受到服务器运行者而非用户的控制。

如果您使用 SaaSS，服务器运行者控制了您的计算。它要求将所有相关数据托付给服务器运行者，该运行者将会被迫将其展示给国家——毕竟，服务器真正是在为谁服务[^6]？

## 原生和次生的不公

如果您使用私有软件或者 SaaSS，您首先是在对您自己作恶，由于这赋予了某些实体凌驾于您之上的不公权力。为了您自己，您应该设法逃脱。如果您作出了关于不分享的承诺，您也是在对他人作恶。恪守这样的承诺是邪恶的，而破坏它则不那么邪恶；为了真正的正直，您完全不应该作出此类承诺。

有这样一些案例，在此，使用非自由软件直接为他人施压以迫使其做类似的事情。Skype 是一个显而易见的例子：如果某人使用非自由的 Skype 客户端软件，它要求另一人也使用该软件——由此两人都牺牲了他们的自由。（Google Hangouts 也有同样的问题。）哪怕只是建议使用这样的程序也是在作恶。我们应该拒绝使用它们，哪怕是短暂地，哪怕是在某些其他人的计算机上。

使用非自由软件或者 SaaSS 的另一种危害在于，这将会奖励加害者，鼓励该程序或者“服务”的进一步开发，进而导致更多人落入商业公司的掌控。

如果其用户是公共实体或者学校，所有这些形式的间接伤害都会被放大。

## 自由软件与国家

公共机构是为民众而非它们自己服务的。当它们进行计算时，它们是在为民众进行计算。它们拥有维护对于该计算的完全控制的职责，以使得它们可以保证该计算是为了民众而妥善进行的。（这构成了国家的计算主权。）它们必须绝不允许对于国家的计算的控制权落入私人之手。

为了维持对于民众的计算的控制，公共机构绝不能利用私有软件（受除了国家以外的其他实体控制的软件）进行该计算。它们也绝不能将其托付给由除了国家以外的其他实体编程和运行的服务，由于这属于 SaaSS。

私有软件在一种关键案例中毫无安全性可言——反对其开发者。而开发者可能会帮助其他人发起攻击。微软在修复 Windows 的 bug 之前将其展示给 NSA[^7]（美国政府的数字间谍机构）。我们不知道苹果是否也做类似的事情，但是它受到与微软相同的政府压力。如果任何其他国家的政府使用这样的软件，这会危害国家安全[^8]。您想要 NSA 闯入您的国家政府的计算机吗？

## 自由软件与教育

学校（这包括所有教育活动）通过它们所讲授的东西而影响社会的未来。它们应该完全讲授自由软件，以便利用其影响行善。讲授私有软件是在植入依赖性，这与教育的任务背道而驰。通过使用自由软件进行训练，学校将会引导社会的未来朝向自由，并且帮助有天赋的程序员精通这项技艺。

它们还将教给学生合作的习惯以帮助他人。每堂课都应该拥有这样的规则：“同学们，这堂课是我们分享知识的地方。如果您将软件带到课堂，您不能自己留着它。与之相反，您必须同课堂上的其他人分享副本——包括源代码，如果某些其他人想要研究它。因此，将私有软件带到课堂是不允许的，除非是为了逆向它。”

私有软件开发者将会迫使我们惩罚那些分享软件的心地善良的学生，并且阻止那些想要修改它的充满好奇心的学生。这意味着恶劣的教育[^9]。

## 自由软件：不止于“优势”

我经常被请求描述自由软件的“优势”。但是当谈到自由的时候，“优势”这个单词太过微不足道。没有自由的生活就是压迫，这也适用于计算以及我们的生活中的其他行为。我们必须拒绝将我们所进行的计算的控制权让给程序或者计算服务的开发者。这是出于自私的原因而做出的正确的事情；但是并非完全出于自私的原因。

自由包括与他人协作的自由。拒绝人们这项自由意味着使人们孤立，这是一种压迫他们的阴谋的开端。在自由软件社区，我们对于合作的自由的重要性非常警觉，由于我们的工作包括有组织的合作。如果您的朋友来访并且看到您使用某个程序，此人可能会请求一份副本。禁止您再分发，或者宣称您“不应该如此做”的程序是反社会的。

在计算中，合作包括向其他用户再分发完整副本。它还包括分发您的修改版本。自由软件鼓励这些形式的合作，而私有软件禁止这些。它禁止再分发副本，并且通过拒绝用户访问源代码，它阻止他们进行修改。SaaSS 拥有同样的效果：如果您的计算是通过网络在某些其他人的服务器上，利用某些其他人的程序副本进行的，您既不能看到也不能接触那些进行了您的计算的软件，于是您不能再分发或者修改它。

## 结论

我们理应拥有对于我们自己的计算的控制；我们如何才能夺回这种控制？通过拒绝我们所拥有或者常规地使用的计算机上的非自由软件，以及通过拒绝 SaaSS。通过开发自由软件[^10]（对于我们当中那些程序员而言）。通过拒绝开发或者推广非自由软件或者 SaaSS。通过将这些思想传播给他人[^11]。

我们和成千上万的用户自 1984 年始已经在如此做，这正是我们现在为何能够拥有任何人——不论是否为程序员——都能够使用的自由的 GNU/Linux 操作系统的原因。作为一位程序员或者活动家加入我们的事业。让我们使得所有计算机用户都获得自由。
